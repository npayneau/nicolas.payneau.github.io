Le lecteur Bluetooth

Les caractéristiques

  • GATT: Generic Attribute Profile
  • Serveur>Service>caractéristique>Valeur
  • Chaque caractéristique peut être lue / écrite / passée en mode notification
  • GATT est l'acronyme de Generic Attribute Profile et définit la manière dont deux appareils Bluetooth Low Energy transfèrent des données dans les deux sens à l'aide de concepts appelés services et caractéristiques
  • utilise un protocole de données générique appelé ATT (Attribute Protocol) , qui est utilisé pour stocker les services, les caractéristiques et les données associées
  • Identifié par des uuid de 16 et 128 bits
  • Le GATT entre en jeu une fois qu'une connexion dédiée est établie entre deux appareils.
  • Un concept important à comprendre avec le GATT est la relation serveur / client.
  • Le périphérique est connu sous le nom de serveur GATT , qui contient les données de recherche ATT et les définitions de service et de caractéristiques, et le client GATT (le téléphone / la tablette), qui envoie des requêtes à ce serveur.

Connaissez-vous le FUGU?

Le projet FUGU

  • Inclu dans le projet Chromium
  • Ouvert à tous les contributeurs et organisations
  • Principaux contributeur Microsoft, Intel, Samsung et Google (entre autres)
  • Expérimental donc ne fonctionne pas partout
L'idée du projet est que tout le monde peut contribué et participer à l'élaboration d'une API. L'idée est d'impliquer les gens et d'intéragir Poisson que si il est bien préparé il sera excelent par contre si mal préparé on peut en mourrir. L'analoge sur les API c'est la même chose. Faire des choses moins bien comme drainer notre baterie

Le projet FUGU

Création d'un process avec itération de feedback: 1. Identification d'un besoin 2. Ecriture d'un semblant de spec (Explainer) 3. Demande de feedback c'est la qu'intervient les contributeurs (nous, Intel et autre) 4. Création de la spec W3C pour spécifier proprement Mise en place de l'origin trial, moyen d'accéder au API expérimental sans rajouter des flags. On peut tester en prod avec un public grace à la souscription de token Origin Trial pour l'API que vous voulez tester Ajout d'un header meta en haut de votre page html.
